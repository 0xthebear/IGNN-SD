Image Resize Utility

./IGNN/utils/imresize.py
▼

This utility file provides a Python implementation of MATLAB's imresize function, specifically for bicubic interpolation.

  • cubic(x): Implements the cubic kernel function used in bicubic interpolation.
  • calculate_weights_indices(...): Computes the interpolation weights and corresponding indices for resizing an image along one dimension. It handles antialiasing and boundary conditions using symmetric padding.
  • imresize(img, scale, antialiasing=True): The main function that takes an image tensor, a scale factor, and an antialiasing flag, and returns the resized image. It performs the resizing independently along the height and width dimensions, handling symmetric padding for boundary pixels.

Graph Construction & Aggregation

./IGNN/models/graph_agg.py
▼

This file contains the core logic for graph construction and aggregation, which is central to the IGNN model. It's based on the N3Net code.

  • compute_distances(xe, ye, I, train=True): Calculates pairwise Euclidean distances between query embeddings (ye) and database embeddings (xe). It can optionally use an index tensor I to restrict the search to a local window, improving efficiency.
  • hard_knn(D, k, I): Performs a hard k-nearest neighbors search, returning the k smallest distances (scores) and their corresponding indices from the distance matrix D. If WITH_WINDOW is enabled, it maps the indices back to the original feature space using I.
  • GraphConstruct(nn.Module): A module responsible for constructing the graph by finding nearest neighbors.
    • It takes xe (son features/database) and ye (father features/query) as input.
    • It converts these features into patches using ops.im2patch.
    • It uses self.indexer (which is index_neighbours) to define a search window for potential neighbors.
    • It then calls self.graph_k to compute distances and find the k nearest neighbors, returning their similarity scores, indices, and optionally difference vectors.
  • GraphAggregation(nn.Module): A module responsible for aggregating features from the identified neighbors.
    • It takes y (query LR features), yd (pixel-shuffled downsampled features of y), and idx_k (indices of k-nearest neighbors) as input.
    • It converts y and yd into patches.
    • self.aggregation gathers features from yd_patch based on idx_k.
    • It can optionally apply adaptive instance normalization (WITH_ADAIN_NROM).
    • Finally, it reconstructs the aggregated features from patches back into an image-like tensor using ops.patch2im, handling scaling for super-resolution.
  • index_neighbours(xe_patch, ye_patch, window_size, scale): Generates index tensors to define local neighborhoods for each query patch. This function is crucial for efficient neighbor search within a limited window and caches results for speed.

Low-Level Operations

./IGNN/models/lib/ops.py
▼

This file provides various low-level operations, including custom CUDA-accelerated functions and patch manipulation utilities.

  • IndexedMatmul1Efficient and IndexedMatmul2Efficient: Custom torch.autograd.Function implementations that wrap CUDA kernels for efficient indexed matrix multiplications. These are critical for performance in graph operations where specific elements need to be accessed and multiplied.
  • indexed_matmul_1_efficient(x,y,I) and indexed_matmul_2_efficient(x,y,I, chunk_size=1024): Convenience functions to call the custom autograd functions.
  • euclidean_distance(x,y): Computes the squared Euclidean distance between two tensors.
  • calc_padding(x_shape, patchsize, stride, padding=None): Calculates the necessary padding for an image before converting it to patches.
  • im2patch(x, patchsize, stride, padding=None, returnpadding=False): Converts an image tensor x into overlapping patches. It uses pyinn.im2col for efficiency.
  • patch2im(x_patch, patchsize, stride, padding): Reconstructs an image from patches, effectively the inverse of im2patch. It uses pyinn.col2im.
  • Im2Patch(nn.Module) and Patch2Im(nn.Module): nn.Module wrappers for im2patch and patch2im.
  • _finfo(tensor) and clamp_probs(probs): Utilities for handling floating-point precision and clamping probabilities.

CUDA Extension Builder

./IGNN/models/lib/setup.py
▼

This setuptools script is used to build the custom CUDA extension matmul_cuda. It compiles the C++ and CUDA source files (matmul.cpp, matmul1_kernel.cu, matmul1_bwd_kernel.cu) into a Python-loadable module, enabling GPU-accelerated indexed matrix operations.

PyTorch–CUDA Interface

./IGNN/models/lib/matmul.cpp
▼

This C++ file acts as the interface between PyTorch and the CUDA kernels. It uses pybind11 to expose the matmul1_cuda and matmul1_bwd_cuda functions (defined in the .cu files) to Python.

CUDA Forward Kernel

./IGNN/models/lib/matmul1_kernel.cu
▼

This CUDA kernel implements the forward pass of the matmul1 operation. It performs an indexed matrix multiplication on the GPU, where mat_i provides the indices for mat_x. This is optimized for parallel execution on CUDA devices.

Training & Testing Orchestrator

./IGNN/core/build.py
▼

This script orchestrates the overall training and testing process.

  • Data Loaders: Sets up data augmentation pipelines (train_transforms, test_transforms) and initializes DataLoader objects for training and validation/testing datasets.
  • Network Initialization: Instantiates the specified SR network (IGNN in this case) and initializes its weights.
  • Optimizer: Configures the Adam optimizer for network training.
  • GPU Support: Wraps the network with torch.nn.DataParallel for multi-GPU training if available.
  • Resume/Load Checkpoint: Handles loading pre-trained weights for resuming training or for testing.
  • Training/Testing Execution: Calls the train function for training or test/test_woGT functions for evaluation based on the cfg.NETWORK.PHASE.
  • Learning Rate Scheduler: Sets up a MultiStepLR scheduler to adjust the learning rate during training.
  • TensorBoard: Initializes SummaryWriter for logging training and validation metrics.

Training Loop

./IGNN/core/train.py
▼

This file defines the main training loop.

  • Epoch Loop: Iterates through a specified number of epochs.
  • Batch Loop: Processes data in batches from the train_data_loader.
  • Loss Calculation:
    • Calculates pixel_loss (e.g., L1 or L2 loss) between the network's output and the ground truth.
    • Optionally calculates perceptualLoss using a VGG19 network if cfg.TRAIN.USE_PERCET_LOSS is enabled.
    • Combines these into total_loss.
  • Optimization: Performs backpropagation (total_loss.backward()) and updates network weights (solver.step()).
  • Metrics and Logging: Tracks batch time, data time, and various losses using net_utils.AverageMeter. Logs these metrics to TensorBoard (train_writer).
  • Validation: Periodically calls the val function (from core/val.py, not provided but implied) to evaluate the model on the validation set.
  • Checkpointing: Saves model checkpoints, including the best performing model based on PSNR.
